DATACENTER

Statement of Work

Install and configure available hardware to support 10-12 Linux and Windows Virtual Machines. Requirements include 3 Proxmox servers configured as a cluster with High Availability. 1 Debian Server for Firewall. 1 Debian Server for Security Event and Incident Manager. 1 CentOS server for file server. Minimum 3 Windows Servers for Active Directory, SQL database, and Web Server. 1 Ubuntu workstation for testing. 1 Kali workstation to test and verify security implementations.

Hardware

3x Dell Optiplex 3070: 3GHz i5-9500 CPU, 32GB DDR4 RAM, 500BG NVMe

1x Dell Optiplex 7470 AiO: 3GHz i5-9500 CPU, 32GB DDR4 RAM, 500BG NVMe

1x Dell Precision T5810: 3.1GHz Xeon E5-1607, 16GB DDR4, 500GB HHD

1x Cisco SG300 switch

1x Netgear PROSafe GS748T switch

Operating Systems and Software

CentOS 7

Process

I started by obtaining my media. I prefer to use Rufus to create my bootable drive. I've found it helps me avoid compatablity issues when it's time to boot the machine and start installations. I know from experience Dell machines have a trick to installing new operating systems and hard drives for that matter. You need to go into the BIOS, open "Sytem Configuration",then "SATA Operation", and switch "RAID On" to "AHCI", save your changes and exit. Your new OS will fail to find the hard drive if you don't. Rebooted, installed ProxMox from USB using default settings. Big mistake on the first one. I assumed the server would pull an IP address from the network via DHCP and did not pay attention to the IP config. Checked my network found a range of available IP addresses. Opened a Nano terminal and updated my IP address in the ETH adapter. Queue the second mistake, because this is a hypervisor I thought I needed a seperate IP for the virtual network and put a second IP in my host file.
